Got fwrite() errors?

If you received fwrite() errors when atempting to backup your Simple Invoices database this means that the webserver user(hopefully your running Apache) doesn't have read/write permissions to the tmp/database_backups directory in the Simple Invoices folder

Please change the permissions on this directory and attempt the backup again. To change the permissions of the tmp/database_backups directory in Unix/Linix/OSX cd to the Simple Invoice directory (cd /var/www/html/simpleinvoices) and then issue the chmod command to give the webserver user read/write permissions (chmod -Rv 777 tmp/database_backups)
